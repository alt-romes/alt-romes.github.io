What you need to know about cabal.project

Multiple Packages

All cabal.project files must have a line stating the packages that comprise the project. For projects with a single package, the first line of the file should resemble:

packages: .

Indicating that the directory the cabal.project file is in (.) contains a .cabal file describing a package.

If more than one package exists in the source tree (e.g. in amazonka.html), all the packages can be specified following packages: on separate lines, for example:

packages:
   examples/
   lib/amazonka/
   lib/amazonka-core/
   lib/amazonka-test/
   lib/amazonka-s3-encryption/

Recent cabal versions also support defining multiple libraries in the same .cabal file.

Dependencies

In the cabal project file one can list additional (external) dependencies, such as patched libraries, libraries only available on a VSC repository, private libraries, etc…

Read Packages Beyond Hackage for notes on depending on packages that are not available in Hackage.

Package Constraints and Flags

An example from ghengin:

package dear-imgui
  flags: +glfw +vulkan -sdl -opengl3

package fir
  flags: +internals

allow-newer:
    dear-imgui:*,
    vector-sized:vector,
    wavefront:base,
    linear-apecs:*,
    typelits-witnesses:*
